Follow this guide to get the codebase running on your local machine

<Warning>
  This documentation assumes you have access to the ProductStack repository on
  GitHub. If you still need to purchase a license, please visit
  getproductstack.com
</Warning>

## Prerequisites

You'll need the following installed locally on your machine:

- [Node.js](https://nodejs.org/en/download/) (>= v20.12.0)
- [Bun](https://bun.sh/docs/installation)
- [Docker](https://www.docker.com)
- [OrbStack](https://www.orbstack.dev/) (recommended)

## Installation Steps

Once you have the met the prerequisites, you can start the development setup.

<Steps>
  <Step title="Clone the repository">

```bash
git clone https://github.com/getproductstack/productstack.git
cd productstack
```

  </Step>
  <Step title="Install dependencies">

```bash
bun install
```

  </Step>
  <Step title="Create the environment file">

```bash
cp .env.example .env
```

Make sure the following environment variables are set:

- `APP_URL`
- `DATABASE_URL`

</Step>

<Step title="Start the development services">

```bash
docker compose up -d
```

</Step>

<Step title="Set up the database">

```bash
bun db:push
```

</Step>

<Step title="Seed the database">

If you would like to modify the account that is created when the database is seeded, you can change the `owner` in `apps/cli/commands/seed/data/account.ts`.

```bash
bun productstack seed
```
</Step>

<Step title="Start the development server">

```bash
bun dev
```
This will start the following services

|                     |                               |
| ------------------- | ----------------------------- |
| Product             | https://localhost:3000         |
| API                 | http://localhost:5432         |
| UI Library         | http://localhost:6100         |
| Email Templates     | http://localhost:3001         |
| Mailhog Server      | http://localhost:8025         |
| Database Studio     | https://local.drizzle.studio  |
| Realtime Server     | ws://localhost:8080           |

<Note>
  Note: The main product is served over https in local development.
</Note> 

</Step>

<Step title="Login to the application">

- Open the [product](https://localhost:3000) `https://localhost:3000` in your browser
- Login with `builder@example.com`, unless you modified the data in step 6.
- Open [Mailhog](http://localhost:8025) `http://localhost:8025` to view the magic link
- Click the magic link in the email to login

</Step>

</Steps>
